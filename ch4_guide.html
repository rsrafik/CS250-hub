üîß DATAPATH COMPONENTS
State Elements
PC: Program counter (address of current instruction)
Instruction Memory: Read-only, no write signal
Register File: 2 read ports, 1 write port
Data Memory: MemRead/MemWrite controls
Pipeline Registers: IF/ID, ID/EX, EX/MEM, MEM/WB
Combinational Elements
ALU: Arithmetic/logic operations
Adders: PC+4, branch target calculation
Sign-Extend: 9-bit ‚Üí 64-bit, 19-bit ‚Üí 64-bit
Multiplexors: ALUSrc, MemtoReg, PCSrc, Reg2Loc
Shift Left 2: Branch offset (word ‚Üí byte)
Key Principle:
No resource can be used more than once per instruction ‚Üí separate instruction & data memories
State elements have clock, combinational elements don't

Data Hazard
Instruction needs data not yet available
Solution: Forwarding, stalling (bubbles)
Control Hazard
Next instruction unknown (branch decision pending)
Solution: Prediction, flushing, stalling

1-Bit Predictor

Bit = 0: Predict not taken
Bit = 1: Predict taken

States:
‚Ä¢ Strongly taken (11)
‚Ä¢ Weakly taken (10)
‚Ä¢ Weakly not taken (01)

‚ö° ILP Concepts
ILP: Instruction-Level Parallelism

Single issue: 1 instruction/cycle
Multiple issue: >1 instruction/cycle (CPI < 1)

Issue slots: Positions to launch instructions
Exploit parallelism by overlapping
instruction execution
üé≤ Speculation

Execute: Speculatively

Commit: If correct, write

üöÄ MULTIPLE ISSUE PROCESSORS
Static Multiple Issue (VLIW)
Compiler decides: Instruction packaging
Issue packet: Multiple ops in wide instruction
Handling hazards: Compiler inserts NOPs
Example: 2-issue LEGv8 (ALU + Load/Store)
Requires: Extra register ports, extra ALU
Dynamic Multiple Issue (Superscalar)
Hardware decides: Issue at runtime
Dynamic scheduling: Reorder execution
Out-of-order exec: Blocked instruction doesn't stall others
Speculation: Guess outcomes, buffer results
üîÅ Loop Unrolling

‚Ä¢ Reduce branch overhead
‚Ä¢ Expose more ILP
‚Ä¢ Enable better scheduling

Drawbacks:
‚Ä¢ Code size increase
‚Ä¢ More registers needed
